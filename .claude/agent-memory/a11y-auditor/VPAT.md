# Southore Design System — VPAT

## WCAG 2.2 Conformance Report

**Product:** Southore Design System
**Version:** 0.0.4
**Date:** 2026-03-15
**Evaluation basis:** Static code analysis of all component `.tsx` and `.css` source files, `src/tokens.css`, and all `src/shared/*.css` files
**Evaluator:** Automated static audit — Accessibility Auditor agent (Claude Sonnet 4.6)
**Conformance target:** WCAG 2.2 Level AA

> **Scope note:** Southore is a React component library, not a deployed application. Criteria that apply to the document/page level (lang attribute, page title, skip navigation, etc.) are the responsibility of the consuming application, not the library. Those criteria are marked Not Applicable and called out in Section 4.

## Section 3: Component Notes

### Badge

No interactive semantics — renders a `<span>`. The circular numeric variant does not automatically provide an accessible label; callers must pass `aria-label`. This is noted in the component interface documentation as an accepted design decision.

### Breadcrumbs

The outer `<nav aria-label="Breadcrumb">` wraps RAC's `<ol>` (which renders as `role="list"`). The current item renders as a plain `<span>` rather than an `<a>` — RAC sets `aria-current="page"` on the current `BreadcrumbItem` via `data-current`. The separator chevron is `aria-hidden`. Pattern is correct.

### Button

Thin wrapper over RAC's Button. No accessible name enforcement — callers of icon-only buttons must supply `aria-label`. The small size (24px height) meets WCAG 2.5.8 minimum but not the recommended 44px for critical actions.

### Calendar / RangeCalendar

RTL-aware (uses `useLocale` to swap chevron direction). Navigation buttons have `aria-label`. Date cells managed by RAC as a grid. Error message connected via `Text[slot="errorMessage"]`. `prefers-reduced-motion` guard present.

### Card

Plain layout component. `titleLevel` defaults to `h3` — callers are responsible for matching the document outline. No interactive role.

### Checkbox / CheckboxGroup

Custom SVG indicator is `aria-hidden`. RAC manages `aria-checked` (including indeterminate). `min-height: 24px` on the label achieves the target-size floor.

### DateFilter

Custom compound component (not a RAC field component). Uses a manual `aria-labelledby`/`aria-describedby` pair — correct, since RAC slot machinery does not apply. The dialog panel has `role="dialog"` with `aria-label="Date filter"`. The standalone `Popover` wraps a dialog div — this is the correct standalone-popover pattern (not a RAC compound component).

### DatePicker / DateRangePicker

RAC compound components. `Popover` contains `Calendar`/`RangeCalendar` without an explicit `<Dialog>` wrapper — this is the correct RAC compound pattern; RAC manages dialog semantics internally. The calendar trigger button in the field group has `aria-label="Open calendar"`.

### Form

Thin wrapper over RAC's `<Form>`. Provides a `display: flex; flex-direction: column` layout.

### Heading

Renders the correct `h1–h6` tag based on `level` prop. Does not enforce heading hierarchy — callers are responsible.

### Link

Thin wrapper over RAC's Link. Inherits full keyboard accessibility and focus ring from RAC + the CSS.

### Menu

Compound component pattern. Section headers rendered via RAC's `<Header>` (correct for group labelling in menus). Separators rendered as `<Separator>` (RAC) which gets `role="separator"`. Destructive item styling uses `data-destructive` attribute driven by the caller.

### NumberField

Increment/decrement buttons receive `aria-label="Increment"` / `aria-label="Decrement"` from RAC automatically via `slot="increment"` / `slot="decrement"`. Stepper button size is 30px × 30px (≥ 24px).

### Popover (standalone)

Wraps RAC's `Popover`. When used as a standalone trigger (not within a RAC compound), callers must supply a `<Dialog>` child for correct dialog semantics — this requirement is documented in the agent MEMORY. DateFilter correctly does this. DatePicker/DateRangePicker/Select/Autocomplete do NOT include a `<Dialog>` wrapper inside Popover — which is correct for RAC compound components.

### RadioGroup / Radio

Pattern matches CheckboxGroup. RAC manages `aria-checked`, group role, and keyboard navigation.

### Select / Autocomplete

RAC compound components. Popover does not need a `<Dialog>` wrapper. Select.Item and Autocomplete.Item render a visually hidden check icon that becomes visible on selection — the `aria-selected` state from RAC is the primary semantic indicator. `textValue` is auto-derived for string children; JSX children require explicit `textValue`. The Autocomplete search field has `aria-label="Search"` and `autoFocus` on popover open.

### Stack

Pure layout utility, `<div>` with no role. No accessibility concerns.

### Switch

RAC manages `role="switch"` and `aria-checked`. The custom track/handle animation respects `prefers-reduced-motion`.

### Tabs

RAC manages the complete tab pattern: `role="tablist"`, `role="tab"` (with `aria-selected`, `aria-controls`), `role="tabpanel"` (with `aria-labelledby`). Arrow key navigation delegated to RAC.

### TextArea

Wraps `<RACTextArea>` inside `<RACTextField>` — correct pairing. The `<textarea>` element itself is a native control, so RAC associates the label via `for`/`id`.

### TextField

Field group (`<Group>`) receives `[data-focus-within]` styles when any inner element is focused, rendering a visible focus ring on the container. The inner `<Input>` also has its own outline suppressed, and focus is visually indicated by the group border change + outline — meeting 2.4.7 and 2.4.11.

### Toast

Uses RAC's experimental (UNSTABLE) Toast API. `ToastRegion` renders `role="region"` with `aria-label="Notifications"`. Individual toasts use RAC's `Toast`/`ToastContent` components. Auto-dismiss at 8000ms (well above WCAG thresholds). Dismiss button has `aria-label="Dismiss notification"`.

### Tooltip

RAC's `TooltipTrigger` with `closeDelay={500}` satisfies 1.4.13 hover persistence. Escape dismissal handled by RAC. The trigger child element must be keyboard-focusable for the tooltip to appear on focus — this is the caller's responsibility but no enforcement exists at the library level.

### ToggleButtonGroup / ToggleButton

RAC manages the toolbar pattern. Buttons within the group receive `aria-pressed`. The group itself does not receive `role="toolbar"` unless callers pass it — RAC's `ToggleButtonGroup` renders with its own role. Size variants meet 2.5.8 (32px and 24px heights).

## Overall Conformance Statement

Southore Design System v0.0.4 **substantially conforms** to WCAG 2.2 Level AA as evaluated by static code analysis. The one area of partial support (1.3.5 Identify Input Purpose) is not a failure of the library itself but a documentation and guidance gap — form input `autocomplete` attributes are caller-supplied and context-specific. All blocking Level A and Level AA criteria are met at the library level for all interactive components audited.

React Aria Components handles the majority of keyboard interaction, ARIA role assignment, and state management. Southore's custom wrappers correctly layer design tokens and visual styles onto those foundations without overriding or conflicting with the accessibility primitives RAC provides.

**Recommended follow-up actions for consuming applications:**

1. Ensure all icon-only interactive elements that wrap library components receive meaningful `aria-label` text at the usage site.
2. Apply `autoComplete` attributes to form fields where input purpose applies (email, name, address, etc.).
3. Provide `<html lang="...">` and `<title>` in every page.
4. Ensure heading levels used via `Card.titleLevel` and `Heading.level` maintain a correct document outline.
5. Ensure Tooltip trigger children are keyboard-focusable elements (native buttons, links, or elements with `tabIndex`).
